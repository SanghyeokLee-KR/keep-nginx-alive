# ⑥ 동맹 관제 · 새벽 불침번 (협업·운영)

> 교육용·방어 전용. 자동화가 못 메우는 빈틈을 **사람·협업**으로 채운 기록. 화면의 서버 IP는 경기 후 인스턴스를 종료해 그대로 둠.

채점은 **5분 간격으로 24시간** 돌아간다. 자동복구(초~1분)가 대부분을 처리하지만, 새벽에 예상 못 한 다운이 길어지면 사람이 자고 있어 놓칠 수 있다. 그래서 **기계(자동복구)** 위에 **이웃(동맹 상호감시)** · **사람(불침번)** · **자동 알림(Teams)** 을 겹쳐 24시간 빈틈을 없앴다.

## 동맹 관제 — 3·5·6조 상호감시

규칙상 **다른 팀 서버가 죽어도 알려줄 의무는 없었다.** 그럼에도 3·5·6조가 **동맹**을 맺어, 서로의 채점 상태를 공유하고 누구든 다운되면 알려주기로 했다. 세 팀의 체커 기록(`t3·t5·t6.md`)을 한 화면에서 **10초마다 크롤링**하고, 어느 팀이든 마지막 체크가 ERROR면 **팀별 고유 알림음**으로 즉시 알린다.

![동맹 관제 — 3·5·6조](../screenshots/alliance-3-5-6-monitor.png)

> Attack/Defense는 본래 경쟁이지만, 우리는 점수보다 **"전원 생존"** 을 함께 노렸다. 상호 가용성 감시 동맹은 그 협업의 산물이다.

## 새벽 불침번 — Discord 교대 감시

자동복구·동맹 감시가 있어도 **사람이 즉시 판단·대응해야 하는 순간**(예: 80 포트 하이재킹 시 수동 stop)이 있다. 그래서 팀원들이 **Discord로 교대표를 짜 새벽 내내 불침번**을 섰다 — 김연동·노현주·이상혁이 시간대를 나눠 번갈아 서버를 지켰다.

![Discord 불침번 교대표](../screenshots/nightwatch-discord.png)

## 자동 알림 — 사람이 못 보는 시간을 메움

불침번이 잠깐 화면을 못 볼 때를 대비해, 상태 변화는 **Teams/webhook으로 자동 푸시**된다(다운→복구·변조 감지·5분 정기 생존보고). 자세한 알림 파이프라인 → [④ 관측·감시](04-monitoring.md).

## 3중 안전망 요약

| 계층 | 누가 / 무엇이 | 역할 |
|---|---|---|
| 기계 | 5중 자동복구 + 헬스체크 자가격리 | 초~1분 내 자동 복구 |
| 이웃 | 동맹 관제 (3·5·6조) | 타 팀 다운까지 상호 감시·알림 |
| 사람 | 새벽 불침번 (Discord 교대) | 자동화가 못 막는 순간 수동 대응 |
| 자동 알림 | Teams / webhook | 사람이 못 보는 시간대 상태 푸시 |

> 방어의 핵심은 스크립트만이 아니었다. **"우리만 살면 된다"가 아니라 "다 같이 살자"** 로 간 협업과, 새벽을 갈아 넣은 불침번이 자동화의 마지막 빈틈을 메웠다.
